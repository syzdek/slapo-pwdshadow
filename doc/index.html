SYNOPSIS

       /tmp/openldap/etc/openldap/slapd.conf

DESCRIPTION

       The pwdshadow overlay translates the attributes  used  by  the  ppolicy
       overlay into attributes compatible with the attributes in the shadowAc‚Äê
       count from RFC2307. The attribtues generated by pwdshadow can  be  used
       instead of the manually maintained attributes of shadowAccount.

CONFIGURATION

       These  slapd.conf configuration options apply to the pwdshadow overlay.
       They shoud appear after the overlay directive.

       pwdshadow_use_pwdpolicy TRUE | FALSE
              Specify that Dereference and use attributes from  the  pwdPolicy
              object to generate pwdShadow attributes. The default is TRUE.

       pwdshadow_default <policyDN>
              Specify  the  DN of the pwdPolicy object to use when no specific
              policy is set on a given user's entry. If there is  no  specific
              policy for an entry and no default is given, then the attributes
              pwdShadowInactive,  pwdShadowMax,  pwdShadowMin,  and   pwdShad‚Ä‚Äê
              owWarning will not be generated.

       pwdshadow_override TRUE | FALSE
              The  generated  values  for pwdShadow attributes can be manually
              overridden by the equivalent shadow attributes. The  default  is
              TRUE.

       pwdshadow_autoexpire TRUE | FALSE
              Generate  the  value  of pwdShadowExpire from pwdMaxAge and pwd‚Ä‚Äê
              GraceExpiry if pwdEndTime is not set on the  user's  entry.  The
              default is FALSE.

OPERATIONAL ATTRIBUTES

       The  operational  attributes used by the pwdshadow module are stored in
       the user's entry. Most of these  attributes  are  not  intended  to  be
       changed  directly  by  users;  they  are used to present the policy at‚Äê
       tributes from the ppolicy module in a format compatible with most  LDAP
       auth  implementations  of  Linux distributions. They have been detailed
       here so that systems administrators can configure their systems to  use
       the generated LDAP attributes.

       userPassword

       The userPassword attribute is not part of the pwdshadow module, however
       it is tracked by the module. Please  refer  to  the  standard  OpenLDAP
       schema for its definition.
